## 1. 课程的组织与目的

实验地点：学院 11 楼实验室。

上课时间：见课表。

课程主要目的：

1. 理解协议在通信中的作用。
2. 掌握抓包软件的开发。
3. 掌握协议解析的编程方法。

## 2. 课程资料

课程提供的资料包括：

1. 关于计算机网络实验的说明。
2. 实验指导文档。
3. WINPCAP（NPCAP）编程文档。

同学还可参考相关编程书籍和网络资料。

## 3. 课程内容与安排

课程实验内容包括：了解抓包工具软件 WIRESHARK、学习 WINPCAP 的编写、编写抓包软件解析 IP 协议。

### 3.1 第一次实验目的与任务

目的：了解抓包工具软件 WIRESHARK 的功能，明确抓包软件的功能。

任务：下载 WIRESHARK，了解其功能。大家可以边上网边通过 WIRESHARK 抓包，查看上网过程中数据包交互的过程，了解通信过程中的协议。

### 3.2 第二次实验目的与任务

目的：学习 WINPCAP 编程。

任务：了解 WINPCAP 的架构，学习其编程接口，编写抓包软件。大家可以下载网络上的相关软件，通过学习了解抓包软件如何编程。

### 3.3 第三次实验目的与任务

目的：分析 IP 协议，统计流量。大家可以把流量计做成流量地图，显示不同地点间数据实时发送的量，比如武汉到北京的数据包发送的数据量。可以用 echart 画地图（或者 D3），用 whois（或者 ip-api）获取 IP 地址的所属地。

任务：根据 IP 协议，解析每个数据包的 PCI，并对不同 IP 进行流量统计。

任务说明：

1. 实现的软件能显示每个包的 PCI。
2. 能显示每个 IP 地址的流量，即显示接收包的数量。

## 4. 考核与提交的报告

1. 第四次实验时，做好的同可以演示。
2. 每位同学应提交程序设计报告。
3. 实验报告内容包括（但不限于）：

实验报告分两部分：

第一部分：

1. 实验的目的。
2. 流量统计软件（流量地图）的功能需求描述。
3. 流量统计软件（流量地图）的设计。
4. 流量统计软件（流量地图）的实现。
5. 总结。

第二部分：

蓝牙定位（测距）系统的设计方案。

4. 实验报告提交形式：电子版文档。
5. 实验报告文件命名：学号 + 姓名 + 班号。
